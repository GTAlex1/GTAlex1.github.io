1. Modelo de Caja (Box Model)

Controles

Espacio interno entre el contenido y el borde.

Espacio externo fuera del borde.

Margin
Padding
Contenido

Profundización Teórica

  • box-sizing: Esta es la propiedad más importante para evitar dolores de cabeza.
    Por defecto es content-box, lo que significa que si pones un ancho de 100px y un padding de 20px, el ancho total será 140px.
    Lo recomendable es usar siempre border-box, para que el padding se incluya dentro del ancho total definido.
  • display: Define cómo se comporta la caja. block ocupa todo el ancho (como un párrafo), inline solo ocupa su contenido (como un enlace) y no acepta márgenes verticales.
  • overflow: Controla qué pasa si el contenido es más grande que la caja. hidden lo corta, scroll añade barras de desplazamiento.

Profundización Teórica

  • La regla de oro del Absolute: Un elemento con position: absolute buscará al ancestro más cercano que tenga posición diferente a static (usualmente relative) para posicionarse. Si no encuentra ninguno, se posicionará respecto a la ventana del navegador.
  • Top, Right, Bottom, Left: Solo funcionan si la posición NO es static. Empujan al elemento desde esa dirección.
  • z-index: Controla la profundidad (eje Z). Piensa en capas de papel. Un número mayor está "más cerca" de tus ojos. z-index solo funciona si el elemento tiene una posición definida (relative, absolute, fixed).

3. Flexbox

Profundización Teórica

  • Los Ejes (Main vs Cross): Flexbox trabaja con dos ejes. El Eje Principal lo define flex-direction.
    Si es row, el eje principal es horizontal. Si es column, el eje principal es vertical.
  • justify-content: SIEMPRE alinea elementos en el Eje Principal.
  • align-items: SIEMPRE alinea elementos en el Eje Cruzado (perpendicular al principal).
  • flex-wrap: Por defecto, Flexbox intenta meter todo en una sola línea (aplastando elementos si es necesario). Usa wrap para permitir que los elementos salten a la siguiente línea si no caben.

Profundización Teórica

  • Diferencia con Flexbox: Flexbox es unidimensional (fila O columna). Grid es bidimensional (filas Y columnas a la vez).
  • Unidad fr: Significa "fracción del espacio disponible". Si tienes 1fr 1fr, cada columna ocupa el 50%. Si tienes 1fr 2fr, la segunda es el doble de ancha que la primera.
  • grid-template-areas: Permite nombrar áreas de tu grilla (ej: "header header" "sidebar content") y colocar elementos llamándolos por su nombre. Es muy útil para maquetar sitios completos ("Layouts").

5. Tipografía

Size
Weight
Line Height
Align

El diseño web es 95% tipografía.

Cuando ajustas el font-size, line-height y weight, cambias completamente la legibilidad y el tono de tu mensaje. Experimenta con estos controles para ver cómo el texto "respira" mejor.

Profundización Teórica

  • font-weight: En CSS moderno usamos números: 400 es Normal, 700 es Bold. Algunas fuentes soportan valores finos como 100 o extra gruesos como 900.
  • line-height: La altura de línea ideal para lectura suele ser entre 1.5 y 1.6 veces el tamaño de la fuente. Si es muy bajo, las líneas se chocan; muy alto, se pierde la continuidad.
  • text-align: justify hace que el texto toque ambos bordes, pero ten cuidado: en web puede crear "ríos" de espacio en blanco difíciles de leer.

9. Transiciones y Transformaciones

Transition (Hover)

Suaviza los cambios de estado. Pasa el mouse sobre el botón.

transition: all 0.5s ease-in-out;

Animation (Keyframes)

Animaciones complejas que se ejecutan solas.

animation: bounce 1s infinite;

Profundización Teórica

  • Diferencia clave:
    - Transition: Necesita un "disparador" (como poner el mouse encima, hacer click, o añadir una clase con JS) para ir del estado A al estado B.
    - Animation: Puede ejecutarse sola al cargar la página, puede tener puntos intermedios (keyframes 0%, 50%, 100%) y puede repetirse infinitamente.
  • Transform: Es la propiedad que mueve, rota o escala el elemento. Es muy eficiente porque usa la tarjeta gráfica (GPU) y no recalcula el diseño de toda la página.

Profundización Teórica

  • Mobile First: Es una filosofía donde diseñas primero para el móvil (el código base) y usas min-width para añadir complejidad en pantallas grandes.
  • max-width: "Aplica este estilo SIEMPRE QUE la pantalla sea MÁS PEQUEÑA que X". Útil para adaptar diseños de escritorio a móvil.
  • min-width: "Aplica este estilo SIEMPRE QUE la pantalla sea MÁS GRANDE que X". Útil para expandir diseños móviles a escritorio.
